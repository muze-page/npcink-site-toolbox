## 一、代码质量修复

### 1. 类名前缀统一（141 处残留）

**问题**：`registry.php` 已使用 `MaBox_` 前缀，但 113 个模块文件中的 `class_exists` 检查和类方法调用仍使用旧 `Npcink_` 前缀，导致类加载时 `class_exists` 检查永远不匹配。

**修复**：
- 批量替换 113 个模块文件中的 `class_exists('Npcink_` → `class_exists('MaBox_`
- 修复 `admin/class-magick-mixtrue-admin.php` 中 12 个 AJAX callback 引用
- 修复模块间方法调用引用（go_top、background_effect、shortcode、template、b2 等）

**影响**：修复后类加载逻辑正常工作，模块可被正确识别和加载。

### 2. 消除 TypeScript as any 滥用（22 处）

**问题**：三个 Vite 项目中大量使用 `as any` 访问 `window.dataLocal`，失去类型安全保障。

**修复**：
- 为三个项目创建 `global.d.ts` 扩展 `Window` 接口
  - `vite/admin/src/global.d.ts` — `DataLocal` 类型
  - `vite/count/src/global.d.ts` — `Receive` 类型
  - `vite/public/src/global.d.ts` — `PublicShareData` 类型
- 添加 `getNonce()` 辅助函数统一 nonce 获取
- feedback/index.tsx tab key 使用 `as const` 断言

**影响**：前端代码获得完整类型安全，编译时可捕获类型错误。

### 3. 处理积压 TODO 项（32 处）

**PHP 修复**：
- `runcode/index.php` — 代码前后换行符问题（添加 `trim()`）
- `single_remove_link.php` — 实现仅移除站外链接，保留站内链接
- `remove_wp_version.php` — 补充样式版本号移除说明
- `bottom_effect/index.php` — 移动端不展示问题添加注释
- `jump_middle_page.php` — 纯链接替换和评论链接待实现转为注释
- `comment_emoji.php` — 评论区判断逻辑已实现
- `wx_xcx_link/custom-template.php` — 小程序内打开页面补充 API 说明

**TypeScript 修复**：
- `poster.tsx` — 实现下载海报功能，补充错误处理
- 其余 29 处 TODO 转为注释说明（设计建议/待实现说明）

## 二、工程化建设

### 4. PHPStan 提升至 Level 3

**变更**：
- Level 1 → Level 3，捕获更多类型错误
- 移除 `public/` 目录扫描（前端构建产物，不应被 PHP 分析）
- 添加 `reportMaybes` 和 `reportStaticMethodSignatures` 配置

### 5. CI 升级 Node.js 到 20 LTS

**变更**：
- Node.js 18 → 20 LTS（18 已于 2025 年 4 月 EOL）
- 覆盖 TypeScript 编译和 Vite 构建任务

### 6. pnpm workspaces monorepo 迁移

**新增**：
- `vite/package.json` — workspace 根配置
- `vite/pnpm-workspace.yaml` — workspace 成员定义

**统一脚本**：
- `pnpm dev:admin / dev:count / dev:public` — 并行开发
- `pnpm build` — 一次性构建所有项目
- `pnpm lint / test` — 统一质量检查

**优势**：
- 共享依赖，减少 node_modules 体积
- 统一版本管理
- 支持跨项目引用（未来可扩展）
