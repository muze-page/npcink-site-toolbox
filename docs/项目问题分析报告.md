## 一、架构与设计问题（高优先级）

| # | 问题 | 位置 | 说明 |
|---|------|------|------|
| 1 | **类名拼写错误** | `includes/class-magick-mixtrue.php` | `Mixtrue` 应为 `Mixture`，文件名、类名、实例化处均受影响，影响可读性和可维护性 |
| 2 | **接口契约形同虚设** | 全部 133 个模块 | `MaBox_Module_Interface` 已定义，但 **0 个模块实际 `implements`** 它。Loader 仅做 `is_subclass_of` 检查并 `error_log`，不阻止加载 |
| 3 | **`run()` 方法签名不统一** | 133 个模块 | 有的 `run()`，有的 `runs()`，有的 `run($config)`，有的 `run($config, $option)`，有的 `run($page_type)`。Loader 通过 `call_user_func` 调用，参数不匹配时静默失败 |
| 4 | **MaBox_Admin 使用 static 一切** | `admin/class-magick-mixtrue-admin.php` | 所有属性和方法都是 `static`，`$plugin_admin` 和 `$plugin_public` 实例化后未被使用，构造函数中的参数赋值给 static 属性，多实例场景会互相覆盖 |
| 5 | **MaBox_Public 完全空壳** | `public/class-magick-mixtrue-public.php` | `load()` 为空，`$magick_mixtrue` 和 `$version` 存为实例属性但从未使用，所有前台功能实际由 admin 模块加载 |
| 6 | **registry 中 option_key 错误** | `admin/modules/registry.php:97` | `optimize.widgets` 的 `option_key` 指向 `optimize.site.hide_email_ip`（与 `optimize.hide_email_ip` 重复），导致两个模块共享同一配置键 |
| 7 | **tiers.php 引用不存在的模块** | `admin/modules/tiers.php:38` | `'privacy.main'` 不在 registry.php 中注册，会导致 Site Health 检测逻辑异常 |
| 8 | **tiers.php 模块重复分类** | `admin/modules/tiers.php:56,57` | `page.go_top` 同时出现在 `core` 和 `advanced` 中 |

## 二、安全问题（高优先级）

| # | 问题 | 位置 | 说明 |
|---|------|------|------|
| 9 | **公开 REST API nonce 验证设计缺陷** | `includes/class-magick-rate-limiter.php:121` | `permission_callback` 将频率限制和 nonce 验证耦合在一起，nonce 验证失败直接返回 `false`（非 WP_Error），客户端无法区分是限流还是 nonce 失效 |
| 10 | **Rate Limiter 依赖 MaBox_Admin** | `includes/class-magick-rate-limiter.php:121` | `MaBox_Rate_Limiter::permission_callback()` 调用 `MaBox_Admin::verify_public_nonce()`，基础设施层依赖业务层，违反依赖方向 |
| 11 | **`$_POST` 直接写入 `$_POST` 属性** | `admin/partials/login/security/login_verify.php:59-65` | 直接修改 `$_POST['sum']`、`$_POST['num1']` 等值，这是超全局变量污染，应使用局部变量 |
| 12 | **SQL 查询未充分防护** | `includes/class-magick-mixtrue-tool.php:387` | `$wpdb->users` 表名直接拼接在 `prepare` 中，虽然用了 `prepare` 但 `{$wpdb->users}` 不在占位符内 |
| 13 | **调试面板无权限检查** | `admin/class-magick-mixtrue-admin.php:110` | `?mabox_debug=1` 仅检查参数存在，不验证用户权限，任何登录用户都能查看路由表信息 |
| 14 | **go 中间页开放重定向风险** | `admin/partials/page/function/go/index.php:4` | `$_GET['url']` 虽经 `esc_url_raw` 处理，但未限制目标域名，可被利用做开放重定向攻击 |
| 15 | **thumbnail_switcher 直接使用 `$_POST`** | `admin/partials/optimize/admin/thumbnail_switcher/easy-thumbnail-switcher.php:161-162` | `$_POST['post_id']` 和 `$_POST['thumb_id']` 未经 `intval`/`sanitize` 直接使用 |

## 三、代码质量问题（中优先级）

| 17 | **MaBox_Tool::p() 调试方法残留** | `includes/class-magick-mixtrue-tool.php:52-57` | `print_r` 调试方法不应存在于生产代码中 |
| 18 | **MaBox_Tool::run_page_hook() 调试方法残留** | `includes/class-magick-mixtrue-tool.php:62-70` | 在后台输出 hook 名的调试功能 |
| 19 | **error_log 散落** | 14 处 | 虽有 `MaBox_Audit_Logger`，但仍有 14 处直接 `error_log()`，未统一走审计日志 |
| 20 | **MaBox_Admin::get_config() 重复实现** | `admin/class-magick-mixtrue-admin.php:810-819` | 与 `MaBox_Helpers::get_config()` 功能重叠 |
| 21 | **`require_once` 全局加载** | `includes/class-magick-mixtrue.php:62-79` | 所有依赖在构造时全部 `require_once`，即使当前请求不需要（如前台请求也加载 admin 类） |

## 四、测试与 CI 问题（中优先级）

| # | 问题 | 位置 | 说明 |
|---|------|------|------|
| 26 | **测试覆盖率极低** | `tests/unit/` | 仅 7 个测试文件，覆盖 ConfigManager、Helpers、Loader、RateLimiter、SvgSanitization、DbCleanDryRun、DatabaseExportSecurity。133 个模块几乎 0 测试 |

| 31 | **uninstall.php 使用 `$wpdb->query("DROP TABLE")`** | `uninstall.php:66` | 直接拼接 DROP TABLE，且未检查表是否存在（虽 IF EXISTS 可兜底，但不符合 WP 规范应用 `dbDelta`） |

## 建议优先处理顺序

### 第一阶段：安全修复（立即处理）

- **#9** REST API nonce 验证设计缺陷 — 分离频率检查和 nonce 验证
- **#11** `$_POST` 污染 — 使用局部变量替代直接修改超全局变量
- **#13** 调试面板无权限检查 — 添加 `current_user_can('manage_options')` 验证
- **#14** go 中间页开放重定向 — 限制目标域名或添加确认页面
- **#15** thumbnail_switcher 输入未过滤 — 添加 `intval()` 和 nonce 验证

### 第二阶段：架构修正

- **#2** 接口契约强制执行 — 让所有模块 `implements MaBox_Module_Interface`
- **#3** `run()` 方法签名统一 — 统一为 `run($config = array())`
- **#6** registry option_key 修正 — 修复 `optimize.widgets` 的重复键
- **#7** tiers.php 清理 — 移除不存在的 `privacy.main` 引用
- **#8** tiers.php 去重 — 移除 `page.go_top` 的重复分类

### 第三阶段：CI 与测试

- **#28** CI 改用 pnpm — 修复 `npm ci` 为 `pnpm install --frozen-lockfile`
- **#30** CI 添加 PHPStan — 在 php-lint 和 php-test 之间添加 phpstan 步骤
- **#26** 测试补充 — 优先覆盖核心类和 REST 端点

### 第四阶段：代码质量

- **#1** 类名拼写修正（需评估破坏性影响）
- **#16** i18n 补全文本域
- **#19** 统一 error_log 为 `MaBox_Audit_Logger`
- **#22** WP_Query 添加 `posts_per_page` 限制
- **#25** 修复 MaBox_Tool 构造函数 bug

### 第五阶段：构建与文档

- **#32** 确认 Vite 构建产物发布策略
- **#35** 移除 demo.html
- **#36** 将 ai/ 目录移出发布范围
- **#40** 更新 `Requires at least` 到实际最低版本
