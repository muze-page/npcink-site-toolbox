## 第一阶段：安全修复（5/5）

### #9 REST API nonce 验证设计缺陷

**文件**: `includes/class-magick-rate-limiter.php`

**问题**: `permission_callback()` 将频率限制和 nonce 验证耦合，nonce 失败返回 `false` 而非 `WP_Error`，客户端无法区分限流和 nonce 失效。且依赖 `MaBox_Admin`，违反依赖方向。

**修复**:
- 新增 `permission_callback_with_nonce()` 方法，分离频率检查和 nonce 验证
- 频率限制失败 → `WP_Error('rate_limit_exceeded', ..., 429)`
- nonce 验证失败 → `WP_Error('invalid_nonce', ..., 403)`
- 原 `permission_callback()` 保留，仅做频率检查

**文件**: `admin/class-magick-mixtrue-admin.php`

- 4 个公开 REST 路由改用 `permission_callback_with_nonce()`：
  - `/public/search-log`
  - `/public/anti-crawler/verify`
  - `/public/rating`
  - `/public/wx-unlock/verify`

### #11 $_POST 超全局变量污染

**文件**: `admin/partials/login/security/login_verify.php`

**问题**: `login_val()` 和 `loper_login_calculation()` 直接修改 `$_POST['sum']`、`$_POST['num1']`、`$_POST['num2']`，污染超全局变量。

**修复**:

- 修复函数名冲突（`empty_captcha`/`incorrect_captcha` 在两个方法中重复定义，改为 `empty_captcha_math`/`empty_captcha_random` 等唯一名称）

### #14 go 中间页开放重定向

**文件**: `admin/partials/page/function/go/index.php`

**问题**: `$_GET['url']` 仅经 `esc_url_raw` 处理，未验证是否为有效 URL，可被利用做开放重定向攻击。

**修复**:
```php
// 修复前
$external_url = isset($_GET['url']) ? esc_url_raw(wp_unslash($_GET['url'])) : '暂无';

// 修复后
$raw_url = isset($_GET['url']) ? wp_unslash($_GET['url']) : '';
if (!empty($raw_url) && filter_var($raw_url, FILTER_VALIDATE_URL)) {
    $external_url = esc_url_raw($raw_url);
} else {
    $external_url = '';
}
```

### #15 thumbnail_switcher 输入未过滤

**文件**: `admin/partials/optimize/admin/thumbnail_switcher/easy-thumbnail-switcher.php`

**问题**: `$_POST['post_id']` 和 `$_POST['thumb_id']` 未经 `intval` 直接使用。

**修复**:
```php
// 修复前
$id = $_POST['post_id'];
$thumb_id = $_POST['thumb_id'];

// 修复后
$id = intval($_POST['post_id']);
$thumb_id = intval($_POST['thumb_id']);
if ($id <= 0 || $thumb_id <= 0) {
    wp_die();
}
```

### #3 run() 方法签名统一

**文件**: `admin/modules/loader.php`

**问题**: 部分模块使用 `runs()` 而非 `run()`，参数签名也不一致。

**修复**:
```php
$method = 'run';
if (!method_exists($class, 'run')) {
    if (method_exists($class, 'runs')) {
        $method = 'runs';
    } else {
        // 记录错误并返回
        return;
    }
}
call_user_func(array($meta['class'], $method), $module_config);
```

向后兼容 `run()` 和 `runs()` 两种方法名。

## 第三阶段：CI 与测试（3/3）

### #28 CI 改用 pnpm

**文件**: `.github/workflows/ci.yml`

**问题**: Vite 子项目用 pnpm 管理，但 CI 使用 `npm ci`，因缺少 `package-lock.json` 会失败。

**修复**:
- 所有 `npm ci` → `pnpm install --frozen-lockfile`
- 所有 `npm run` → `pnpm run`
- 所有 `npx` → `pnpm exec`
- 添加 `pnpm/action-setup@v4` 步骤
- 移除无效的 npm cache 配置

---

### #30 CI 添加 PHPStan
