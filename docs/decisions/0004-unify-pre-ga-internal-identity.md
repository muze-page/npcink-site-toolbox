# ADR-0004: 首次发布前统一内部标识与存储键

## 状态

已接受

## 日期

2026-07-18

## 背景

ADR-0003 完成了 Npcink Site Toolbox 的公开身份统一，但为了把公开品牌迁移与内部重构分开，暂时保留了 `MaBox_*` 类、`MAGICK_MIXTURE_*` 常量和 `Magick_ToolBox_*` Option key。

项目目前仍未正式发布，没有真实用户、第三方扩展调用方或必须保留的安装数据。继续保留多套前缀会让代码搜索、堆栈排障、自动加载、数据库检查、卸载清理和开发文档长期承担额外映射成本；等公开发布后再处理，反而需要类别名、双读、数据迁移和回滚逻辑。

因此，ADR-0003 中“保留内部标识和存储键”的范围控制已经完成其阶段性作用，但不再是当前成本最低的长期方案。

## 决策

- 主运行类使用 `Npcink_Site_Toolbox`，其他插件类和接口统一使用 `Npcink_Toolbox_*`。
- 全局常量统一使用 `NPCINK_SITE_TOOLBOX_*`。
- 核心 PHP 文件统一使用 `class-npcink-site-toolbox.php`、`class-npcink-toolbox-*.php` 和 `interface-npcink-toolbox-*.php` 命名，并同步自动加载及发布包契约。
- 插件自有 Option、Transient、对象缓存、Cookie、LocalStorage、Widget ID 和评论元数据统一使用 `npcink_site_toolbox_*`；模块配置固定为五个顶层 Option，加上独立运行数据。
- 不保留旧类别名、旧 Option 双读、迁移器或兼容清理。Local 开发站的数据直接重置为新键。
- `uninstall.php` 只清理当前正式运行面使用的新键；从未公开发布且已经退役的旧表、旧模块和旧数据键不进入产品兼容代码。
- ADR、历史发布记录和 `ai/reference` 快照保留当时的原始命名，避免改写证据。
- 纯渲染层的私有 DOM/CSS 选择器不属于 PHP 或持久化契约，本次不做无收益的全量改名；需要调整时应放在独立界面工作包中验证。

本决策只取代 ADR-0003 中“保留内部类名、常量名、Option key 和其他内部存储标识”的条款。ADR-0003 对公开名称、slug、文本域、REST namespace、发布包和历史记录的决定继续有效。

## 理由

当前是消除内部命名债务的最后低成本窗口。没有用户数据意味着不需要编写和永久维护迁移逻辑；没有第三方调用方意味着 PHP 类和 Hook 回调可以一次性切换。统一后的代码、数据库和卸载逻辑都直接表达当前产品身份，减少后续开发和排障成本。

重构仍按两个独立工作包实施：先完成 PHP 标识与加载路径，再完成持久化和跨请求键。这样可以分别定位自动加载回归与数据契约回归，避免一次盲目替换所有 `magick` 或 `mabox` 文本。

## 备选方案

### 永久保留旧内部名称

拒绝。它避免了一次机械重构，但会把名称映射、数据库辨认和文档解释成本永久留给后续开发。

### 为旧名称增加兼容别名和 Option 迁移

拒绝。当前没有用户和已发布安装，兼容层没有服务对象，只会扩大运行路径和测试矩阵。

### 同时引入 PHP Namespace、PSR-4 和新目录架构

拒绝。本阶段目标是消除旧身份和存储前缀，不是重写模块架构。现有显式自动加载和静态模块契约仍能满足产品边界；同时改变架构会让回归原因更难定位。

### 连 DOM/CSS 选择器一起全量改名

拒绝。私有选择器不形成数据库或扩展契约，全量改名主要制造前端构建和视觉回归面，维护收益不足。

## 影响

- 旧开发数据库中的设置、日志、限流状态和 LocalStorage 不会被自动迁移；开发站需要清理旧键并重新保存设置。
- PHP 堆栈、模块 Registry、自动加载映射和发布包文件名只出现当前 Npcink 标识。
- 新增身份契约测试会扫描正式 PHP 运行面，阻止旧类、旧常量和旧存储键重新进入发布代码。
- 发布前必须通过 PHPUnit、PHPStan、前端构建、发布 ZIP 和真实 WordPress 激活/设置保存冒烟。

## 回滚

如果 PHP 标识工作包出现问题，可整体回退类名、文件名、Registry 和自动加载提交；如果存储工作包出现问题，可整体回退新键提交并恢复 Local 数据快照。不得通过临时类别名、双读或同时写入两套 Option 来掩盖未定位的回归。
