## 二、P0 整改详情

### 2.1 REST API 参数 schema 补全

**问题**：约 40% 的 REST 端点缺少 `args` 定义，参数验证仅在回调函数内手动处理。

**整改**：为以下端点补充完整的 `args` 配置（含 `validate_callback` + `sanitize_callback`）：

| 端点 | 新增参数 |
|------|----------|
| `POST /settings` | `modules`（object）、`global`（object） |
| `POST /performance/media/check` | `post_id`（int, 可选） |
| `POST /performance/seo/fix-alt` | `post_id`（int, 可选） |
| `POST /page/batch-replace` | `pairs`（array，含 search/replace 验证 + wp_kses_post 清洗）、`dry_run`（bool） |
| `POST /page/batch-replace/rollback` | `confirm`（bool, 必填，防止误操作） |

**同时清理了 3 对重复路由注册**：
- `/performance/media/fix-alt`（重复 2 次）
- `/performance/seo/check`（重复 2 次）
- `/domestic/baidu/push`（重复 2 次）

**文件变更**：`admin/class-magick-mixtrue-admin.php`

### 2.2 核心安全逻辑 PHPUnit 测试

**问题**：验收标准要求"PHPUnit 测试覆盖核心安全逻辑"，但原有 3 个测试文件均不涉及安全。

**新增 4 个测试文件，共 50+ 条测试用例**：

| 测试文件 | 覆盖内容 | 用例数 |
|----------|----------|--------|
| `RateLimiterTest.php` | 限流器类存在性、方法签名、默认配置、transient 存储、封禁逻辑 | 12 |
| `SvgSanitizationTest.php` | 危险标签移除（script/object/embed/iframe/form/link/meta/base）、危险属性移除（onclick/javascript:/vbscript:/expression）、XXE 防护（DOCTYPE/ENTITY）、安全内容保留 | 18 |
| `DatabaseExportSecurityTest.php` | 敏感字段检测（pass/secret/token/key）、脱敏行为（邮箱/IP/密码/长字符串）、表名白名单、行数限制、SQL 注入防护 | 16 |
| `DbCleanDryRunTest.php` | dry-run 默认值验证、REST API 配置、batch replace 备份/回滚机制 | 10 |

**文件变更**：新增 `tests/unit/RateLimiterTest.php`、`SvgSanitizationTest.php`、`DatabaseExportSecurityTest.php`、`DbCleanDryRunTest.php`

## 三、P1 整改详情

### 3.1 #12 统一审计日志中心

**问题**：各模块使用 `error_log()` 散乱记录日志，格式不统一，无法查询和分析。

**方案**：新增 `MaBox_Audit_Logger` 类，提供结构化审计日志能力。

**核心设计**：

```
日志级别：info / warning / error / critical
日志类别：security / database / config / api / file / auth / rate_limit
存储方式：error_log（默认）+ 数据库可选（MABOX_ENABLE_AUDIT_LOG_STORAGE）
扩展机制：mabox_audit_log 动作钩子，允许第三方监听
```

**便捷方法**：
- `MaBox_Audit_Logger::security()` — 安全事件
- `MaBox_Audit_Logger::database()` — 数据库操作
- `MaBox_Audit_Logger::config()` — 配置变更
- `MaBox_Audit_Logger::api_error()` — API 错误
- `MaBox_Audit_Logger::file()` — 文件操作
- `MaBox_Audit_Logger::rate_limit()` — 限流事件

**已集成模块**：

| 模块 | 日志类别 | 记录内容 |
|------|----------|----------|
| 频率限制器 | rate_limit | 触发 IP+UA、请求次数、时间窗口 |
| SVG 上传 | file | 文件名、操作用户 |
| 数据库导出 | database | 表名、用户、导出行数 |
| 批量替换 | database | 替换文章数、用户 |
| 批量回滚 | database | 回滚文章数、用户 |
| 数据库清理 | database | 清理类型、dry_run 状态、用户 |
| 配置保存 | config | 配置更新事件、用户 |

**文件变更**：新增 `includes/class-magick-audit-logger.php`，修改 6 个模块文件

### 3.2 #9 功能分层

**问题**：90+ 功能未区分风险等级，用户无法判断哪些功能安全、哪些需要谨慎使用。

**方案**：新增分层定义文件 `admin/modules/tiers.php`，将模块分为四级：

| 层级 | 数量 | 定义 | 示例 |
|------|------|------|------|
| `core` | ~65 | 核心功能，稳定、安全、推荐使用 | SEO 优化、评论增强、备案合规、百度推送 |
| `advanced` | ~4 | 进阶功能，正常使用需注意兼容性 | 底部特效、广告位 |
| `high_risk` | 10 | 高风险功能，谨慎启用 | 禁止更新、禁止复制、CDN 替换、批量替换 |
| `experimental` | ~18 | 实验性功能，不稳定或特定场景 | 点击特效、屏幕毛发、像素鸡、H5 移动端 |

**Loader 新增 API**：
- `MaBox_Module_Loader::get_tiers()` — 获取全部分层定义
- `MaBox_Module_Loader::get_module_tier($module_id)` — 获取指定模块层级

**文件变更**：新增 `admin/modules/tiers.php`，修改 `admin/modules/loader.php`

### 3.3 #10 WordPress Site Health 集成

**问题**：用户无法在 WordPress 原生界面中了解插件运行状态和潜在风险。

**方案**：新增 `MaBox_Site_Health` 类，在「工具 > 站点健康」中添加 7 项检测：

| 检测项 | 类型 | 检测内容 | 状态判断 |
|--------|------|----------|----------|
| PHP 版本 | direct | 当前版本 >= 7.4 | good / critical |
| WordPress 版本 | direct | 当前版本 >= 6.0 | good / recommended |
| 伪静态 | direct | permalink_structure 非空 | good / recommended |
| 对象缓存 | direct | wp_using_ext_object_cache() | good / recommended |
| REST API 可用性 | direct | 实际请求 REST 端点 | good / critical |
| 已激活模块数 | direct | 激活数 / 总数 | good |
| 高风险模块检查 | direct | 统计 high_risk + experimental 激活数 | good / recommended / critical |

**文件变更**：新增 `includes/class-magick-site-health.php`，修改 `includes/class-magick-mixtrue.php`
