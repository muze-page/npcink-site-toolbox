# WP Magick Toolbox v2.4 稳定性整改计划

> **版本**: 2.4.0  
> **目标**: 从"功能集合"升级为"稳定、安全、可恢复、可诊断的站长工具平台"  
> **原则**: v2.4 不加大功能，只做安全、权限、回滚、日志、限流、文档

## 二、核心问题分析

### 2.1 安全风险（最高优先级）

| 风险点 | 位置 | 严重性 | 说明 |
|--------|------|--------|------|
| REST API 缺少参数 schema | `class-magick-mixtrue-admin.php:410-549` | 高 | 所有端点无 `validate_callback` / `sanitize_callback` |
| 数据库导出无敏感字段脱敏 | `download-sql-table.php:57-121` | 高 | 可导出任意表，含用户密码、邮箱等 |
| 数据库清理无 dry-run | `db_clean/index.php:46-80` | 高 | 直接删除，无预览确认 |
| 批量替换无回滚 | `batch_replace.php:39-76` | 高 | 不可逆操作，无内容快照 |
| SVG 上传无清洗 | `svg_support.php:23-26` | 高 | 允许原始 SVG，含 XSS 风险 |
| Public API 无限流 | `class-magick-mixtrue-admin.php:520-545` | 中 | 搜索日志、评分、验证接口无频率限制 |

### 2.2 缓存失效风险

- `active_modules` 对象缓存 TTL 1 小时
- 配置保存后已清除缓存（`clear_config_cache()`），但以下场景未覆盖：
  - 主题切换（`switch_theme`）
  - 插件升级（`upgrader_process_complete`）
  - 导入配置后
  - 多站点切换

### 2.3 性能风险

- 90+ 功能中大量前端特效（点击特效、背景特效、屏幕毛发等）
- 移动端 LCP / CLS 可能受影响
- 部分功能使用率低但持续加载

### 2.4 维护风险

- 第三方依赖（B2 主题、百度 API、微信 JSSDK、OSS）变动频繁
- 无自动化回归测试覆盖核心模块
- 模块间静态调用耦合

## 四、P0 实施详情

### 任务 1：REST API 全量权限审计

**现状**：所有端点已有 `permission_callback`，但部分使用 `__return_true` + nonce 验证。

**整改**：
- 保留现有权限检查
- 公开接口（`/public/*`）增加频率限制（任务 7）
- `/tools/categories` 公开接口保持现状（仅返回分类名称，无风险）

**涉及文件**：
- `admin/class-magick-mixtrue-admin.php`

### 任务 2：REST API 参数 schema 化

**现状**：所有端点无 `args` 定义，参数验证在回调函数内手动处理。

**整改**：为每个端点添加 `args` 配置，包含：
- `required`：是否必填
- `sanitize_callback`：清洗函数
- `validate_callback`：验证函数

**涉及端点**：
| 端点 | 关键参数 |
|------|----------|
| `/settings` (POST) | 完整配置对象 |
| `/performance/db/clean` | `type`（清理类型） |
| `/page/batch-replace` | `pairs`（替换规则）、`dry_run` |
| `/tools/table-data` | `databaseName`（表名） |
| `/public/search-log` | `keyword`（搜索词） |
| `/public/rating` | `post_id`、`score` |
| `/public/anti-crawler/verify` | `ticket`、`randstr` |
| `/ai-review/*` | 各模块参数 |
| `/feedback/*` | 反馈内容 |
| `/ticket/*` | 工单内容 |

**涉及文件**：
- `admin/class-magick-mixtrue-admin.php`
- 各模块 `index.php` 中的 `register_rest_routes` 方法

### 任务 3：数据库导出限制

**现状**：允许导出任意表名（通过正则白名单验证），无敏感字段脱敏。

**整改**：
1. 默认只允许导出插件自有表
2. 敏感字段自动脱敏（email、phone、password、token、secret）
3. 大表分页导出（限制 1000 行/次）
4. 导出行为写入审计日志
5. 仅超级管理员可操作

**涉及文件**：
- `admin/partials/function/download-sql-table.php`

### 任务 4：数据库清理 dry-run

**现状**：直接执行 DELETE 操作，无预览。

**整改**：
1. 添加 `dry_run` 参数（默认 true）
2. dry_run 模式返回影响行数，不执行删除
3. 执行模式需要显式传递 `dry_run=false`
4. 清理前记录操作日志

**涉及文件**：
- `admin/partials/performance/db_clean/index.php`

### 任务 5：批量替换加回滚

**现状**：直接替换文章内容，无回滚机制。

**整改**：
1. 添加 `dry_run` 参数（默认 true）
2. dry_run 模式返回受影响文章列表
3. 执行前备份原始内容到 `wp_postmeta`（key: `_mabox_backup`）
4. 提供回滚接口恢复备份内容

**涉及文件**：
- `admin/partials/page/function/batch_replace.php`

### 任务 6：SVG 安全处理

**现状**：直接允许 SVG 上传，无清洗。

**整改**：
1. 仅管理员可上传 SVG
2. 添加 SVG 清洗（移除 `<script>`、`<object>`、`<embed>` 等危险标签）
3. 或使用 `wp_kses` 过滤 SVG 内容

**涉及文件**：
- `admin/partials/optimize/medium/svg_support.php`

### 任务 7：Public API 限流

**现状**：公开接口仅依赖 nonce 验证，无频率限制。

**整改**：
1. 基于 IP + User Agent 组合限流
2. 默认限制：60 次/分钟
3. 使用 transient 存储计数
4. 超限返回 429 状态码

**涉及文件**：
- `admin/class-magick-mixtrue-admin.php`
- 新增 `includes/class-magick-rate-limiter.php`

### 任务 8：外部服务隐私说明

**现状**：连接多个外部服务，无明确隐私说明。

**整改**：
1. 在后台添加"隐私与外部服务"说明页
2. 列出所有外部服务及其用途
3. 明确哪些数据会出站
4. 匿名统计默认关闭（opt-in）

**涉及文件**：
- 新增 `admin/partials/privacy/index.php`
- 前端添加隐私说明页面

## 五、执行顺序

```
任务 1（权限审计） → 任务 2（参数 schema） → 任务 7（Public API 限流）
                                                    ↓
任务 8（隐私说明） ← 任务 6（SVG 安全） ← 任务 5（批量替换回滚） ← 任务 4（DB dry-run） ← 任务 3（DB 导出限制）
```

**推荐执行顺序**：3 → 4 → 5 → 6 → 7 → 1 → 2 → 8

理由：先修复最高风险功能（数据库导出、清理、批量替换），再做安全加固（SVG、限流），最后做审计和文档。

## 六、风险与缓解

| 风险 | 影响 | 缓解 |
|------|------|------|
| REST API schema 化可能破坏前端 | 高 | 保持现有参数名，只添加验证层 |
| 数据库导出限制可能影响用户 | 中 | 提供白名单配置，允许管理员添加信任表 |
| 批量替换回滚增加存储 | 低 | 回滚数据在下次替换或手动清理时删除 |
| Public API 限流可能误伤 | 中 | 限流阈值可配置，日志记录误判 |

## 七、验收标准

- [ ] 所有 REST API 端点通过 PHPStan 检查
- [ ] 所有高风险操作有 dry-run 模式
- [ ] 数据库导出无法获取敏感字段明文
- [ ] Public API 限流生效（可通过测试验证）
- [ ] SVG 上传后不含危险标签
- [ ] 批量替换可回滚
- [ ] 隐私说明页面可访问
- [ ] PHPUnit 测试覆盖核心安全逻辑
