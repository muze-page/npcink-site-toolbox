# 数据库清理

::: tip 推荐
定期清理。保持数据库健康。
:::

## 功能说明

清理 WordPress 数据库中的垃圾数据，包括修订版本、草稿、垃圾评论、临时数据等，减少数据库体积，提升查询性能。

## 可清理的数据类型

| 类型 | 说明 | 影响 |
|------|------|------|
| 修订版本 | 文章历史修订记录 | 删除后无法恢复历史版本 |
| 自动草稿 | 自动保存的草稿 | 无影响 |
| 垃圾评论 | 标记为 spam 的评论 | 无影响 |
| 临时数据 | Transient 缓存 | 会自动重建 |
| 待审评论 | 待审核的评论 | 谨慎清理 |
| 回收站 | 回收站中的内容 | 永久删除 |

## 使用场景

- 数据库体积过大，查询变慢
- 修订版本积累过多
- 大量垃圾评论占用空间
- 定期维护数据库

## 开启方法

1. 进入 **魔法工具箱** → **国内生态**
2. 开启「数据库清理」
3. 选择要清理的数据类型
4. 查看预估清理数据量
5. 确认清理

## 配置选项

| 选项 | 说明 | 推荐 |
|------|------|------|
| 清理修订版本 | 删除文章历史修订 | ✅ |
| 清理草稿 | 删除自动草稿 | ✅ |
| 清理垃圾评论 | 删除 spam 评论 | ✅ |
| 清理临时数据 | 删除过期 transient | ✅ |
| 自动清理 | 定时自动清理 | - |
| 清理周期 | 自动清理频率 | 每周 |

## 注意事项

- **清理前务必备份数据库**
- 修订版本删除后无法恢复
- 建议先查看预估清理数据量
- 自动清理功能适合定期维护
- 清理操作不可逆，请谨慎操作

## 性能影响

- 清理修订版本可减少 50%+ 的 post 表数据
- 清理垃圾评论可减少 comment 表 30%+ 数据
- 清理后建议优化数据库表
